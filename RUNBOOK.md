# 📖 운영 매뉴얼 (Runbook)

이 문서는 개발자가 아닌 운영자 및 팀장님이 서비스를 관리하는 방법을 설명합니다.

## 1. 랜딩 페이지 콘텐츠 수정 (Sanity)
코드를 수정하지 않고 웹에서 즉시 텍스트와 이미지를 변경할 수 있습니다.

1. **접속:** Sanity Studio 주소(예: `https://kobaco-atv.sanity.studio`)로 접속합니다.
2. **수정:** `HomePage` 문서를 선택합니다.
3. **항목:** 
   - **Hero:** 메인 제목, 소제목, 버튼 문구 수정 가능.
   - **Sections:** 하단의 소개 섹션들을 추가, 삭제하거나 드래그하여 순서를 바꿀 수 있습니다.
4. **반영:** 수정 후 **"Publish"** 버튼을 누르면 약 1분 내외로 실제 웹사이트에 반영됩니다.

## 2. 광고 정책 및 단가 수정 (Policy)
매체별 단가, 보너스율 등 핵심 로직 데이터를 수정하는 방법입니다.

이 데이터는 실수 방지를 위해 **Git 파일**로 관리됩니다.
1. `/policy` 폴더 내의 파일을 엽니다.
   - `channels.json`: 매체별 기본 CPV 단가
   - `bonuses.json`: 기본/기간/볼륨/프로모션 보너스율
   - `surcharges.json`: 지역 할증 등 할증률
2. 값을 수정한 뒤 저장합니다.
3. Git 커밋 후 푸시하면 Vercel에 의해 자동으로 재배포 및 반영됩니다.

## 3. 사용 히스토리 확인 (선택 사항)
사용자가 견적 기능을 어떻게 사용했는지 확인하려면:
1. Vercel 대시보드에서 **KV (Storage)** 기능을 활성화해야 합니다.
2. 활성화된 경우 `/admin/usage` 페이지(추후 구현 예정) 또는 Vercel KV CLI에서 직접 로그를 확인할 수 있습니다.

## 4. 장애 대응
- **사이트 접속 불가:** Vercel Dashboard의 'Deployments' 탭에서 마지막 성공 버전으로 'Rollback' 하세요.
- **계산 결과 이상:** `/policy` 폴더의 JSON 파일 오타(콤마 등)를 확인하세요.
